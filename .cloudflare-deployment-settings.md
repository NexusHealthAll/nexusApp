# Cloudflare Pages Deployment Settings

## Current Issue
The deploy command in Cloudflare Pages is set to `npx wrangler deploy`, but for Pages projects it should be `npx wrangler pages deploy` or just remove the deploy command entirely (Cloudflare Pages will auto-deploy the build output).

## Solution Options

### Option 1: Update Deploy Command (Recommended)
In Cloudflare Pages Dashboard → Settings → Builds & deployments:
- **Deploy command**: Change to `npx wrangler pages deploy dist`

### Option 2: Remove Deploy Command (Simplest)
In Cloudflare Pages Dashboard → Settings → Builds & deployments:
- **Deploy command**: Leave **empty** or remove it entirely
- Cloudflare Pages will automatically deploy the `dist` folder after build

### Option 3: Use --assets flag
- **Deploy command**: `npx wrangler deploy --assets=./dist`

## Current Configuration
- **Framework**: Vite
- **Build command**: `npm run build`
- **Output directory**: `dist`
- **Node version**: 22.16.0

## Notes
- Build is working correctly with Vite 6.4.3
- The issue is only with the deployment step
- Cloudflare Pages doesn't require wrangler for deployment - it can auto-deploy static files
